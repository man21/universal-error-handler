# Contributing to Universal Error Handler 🤝

First off, thank you for considering contributing to **Universal Error Handler**! It's people like you who make this a great tool for the entire JavaScript community.

## 🚀 How Can I Contribute?

### Reporting Bugs
*   Check the [Issues](https://github.com/man21/universal-error-handler/issues) to see if the bug has already been reported.
*   If not, open a new issue. Include a clear title, a description of the problem, and steps to reproduce the bug.
*   Provide your environment details (Node.js version, OS, etc.).

### Suggesting Enhancements
*   Open an issue with the tag `enhancement`.
*   Explain why this feature would be useful and how it should work.

### Your First Code Contribution
1.  **Fork** the repository.
2.  **Clone** your fork: `git clone https://github.com/YOUR_USERNAME/universal-error-handler.git`
3.  **Create a branch** for your changes: `git checkout -b feat/my-new-feature`
4.  **Install dependencies**: `npm install`
5.  **Make your changes**. Ensure you follow the existing coding style.
6.  **Run tests**: `npm test`
7.  **Commit your changes**: `git commit -m "feat: added a cool new feature"`
8.  **Push to your fork**: `git push origin feat/my-new-feature`
9.  **Submit a Pull Request** (PR) to the `master` branch.

## 📜 Coding Guidelines
*   Keep functions small and focused.
*   Write meaningful variable and function names.
*   Ensure all new features include unit tests in the `tests/` directory.
*   Update the `README.md` if you add or change functionality.

## ⚖️ Code of Conduct
By participating in this project, you agree to abide by our Code of Conduct. Be respectful, inclusive, and professional.
